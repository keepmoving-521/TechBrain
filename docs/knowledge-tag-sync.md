# Markdown 文档标签同步说明

本文档对应 TB-V10-021：同步 Markdown 文档标签。

## 1. 同步目标

Markdown Front Matter 的 `tags` 是文档标签的事实来源。每次文档同步都会把标签数组映射为：

- `knowledge_tags` 中的规范化标签实体。
- `knowledge_document_tags` 中的文档标签关联。
- `knowledge_documents.tags` 中的源标签字符串数组镜像。

```text
Front Matter tags
        ↓
名称校验与规范化
        ↓
创建或复用 knowledge_tags
        ↓
替换 knowledge_document_tags
```

## 2. 同步规则

对于每个 Front Matter 标签，系统执行：

1. Unicode NFKC 字符规范化。
2. 去除首尾空白并合并连续空白。
3. 使用 Unicode `casefold` 生成规范化唯一键。
4. 校验名称和规范化名称长度不超过 80 个字符。
5. 按 `normalized_name` 查找已有标签。
6. 不存在则创建，存在则复用。
7. 如果已有标签为 `archived`，重新激活为 `active`。

例如：

```yaml
tags:
  - ORM
  - 性能优化
```

若数据库已有规范化名称为 `orm` 的标签，则直接复用，不创建新的 `ORM` 标签。

## 3. 关联替换语义

文档结构化标签始终精确匹配当前 Front Matter 标签集合。

原内容：

```yaml
tags:
  - orm
  - sqlalchemy
```

修改后：

```yaml
tags:
  - ORM
  - performance
```

同步结果：

- 复用规范化名称为 `orm` 的标签。
- 新建或复用 `performance`。
- 删除文档与 `sqlalchemy` 的关联。
- 保留 `sqlalchemy` 标签实体，避免影响其他文档和历史管理数据。

当 Front Matter 使用：

```yaml
tags: []
```

文档的全部结构化标签关联会被清空，但标签实体不会被自动删除。

## 4. 新增、修改与恢复

标签同步接入统一文档同步流程：

| 文档场景 | 标签处理 |
| --- | --- |
| 首次发现 | 创建或复用标签，并建立全部关联 |
| 正文或元数据修改 | 将关联替换为最新标签集合 |
| 标签增加 | 新增缺失关联 |
| 标签删除 | 删除多余关联 |
| 文件移动 | 保持并重新校验标签关联 |
| 软删除 | 保留标签关联，普通查询通过文档删除状态过滤 |
| 文档恢复 | 按恢复后的 Front Matter 重新建立准确关联 |

## 5. 幂等与自动修复

重复同步同一文档时：

- 不重复创建标签。
- 不重复创建文档标签关联。
- 标签和关联均正确时，文档仍返回 `unchanged`。

即使 Markdown 哈希没有变化，只要结构化标签关联缺失或不一致，同步也会修复关联并返回 `updated`。这使升级到 `0007` 后的已有文档可以通过一次全量同步完成标签关联回填，无需单独的数据迁移脚本。

## 6. 重复与非法标签

同一文档内不允许两个标签在规范化后相同。

以下写法会被拒绝：

```yaml
tags:
  - ORM
  - orm
```

即使字符外观不同，NFKC 规范化后相同也会被拒绝，例如半角和全角 ORM。

同步错误码：

| 错误码 | 说明 |
| --- | --- |
| `TAG_INVALID_NAME` | 标签为空、过长或包含控制字符 |
| `TAG_DUPLICATE_NORMALIZED_NAME` | 同一文档内标签规范化后重复 |

错误记录包含文档路径和 `tags` 字段，不写入文档、分类、标签或关联的半成品数据。

## 7. 一致性边界

- Markdown `tags` 决定文档当前应关联哪些标签。
- 标签展示名称首次创建后保持稳定；后续大小写变化不会静默重命名已有标签。
- 标签同义词和近似词不自动合并，由 TB-V10-024 显式执行标签合并。
- 未被任何文档使用的标签不会在同步时自动删除，由后续标签管理功能处理。

## 8. 验收结论

TB-V10-021 已覆盖：

- 首次同步根据 Front Matter 创建标签并建立多对多关联。
- 重复同步不产生重复标签或重复关联。
- 新增、删除和修改文档标签后，结构化关联精确更新。
- 规范化名称相同的标签稳定复用。
- 已归档但重新出现在 Markdown 中的标签自动激活。
- 哈希未变化但关联缺失时可以自动修复。
- 非法或规范化后重复的标签返回明确错误。
