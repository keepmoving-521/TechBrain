# 分类知识列表页

本文档对应 TB-V10-027：实现分类知识列表页。

## 1. 页面入口

```text
/knowledge
```

页面由左侧分类树和右侧文档列表组成。首页的常用分类入口使用以下地址直接选中分类：

```text
/knowledge?category_id={category_id}
```

页面刷新后会从 URL 恢复当前分类、页码和排序方式。

## 2. 数据接口

页面复用已有接口：

```http
GET /api/v1/categories/tree
GET /api/v1/documents
```

分类树只展示 `active` 分类，保留后端返回的层级和稳定排序。文档列表默认只展示 `published` 与 `deprecated` 文档。

## 3. 查询状态

| URL 参数 | 默认值 | 说明 |
| --- | --- | --- |
| `category_id` | 无 | 当前直属分类 ID；无值时浏览全部知识 |
| `page` | `1` | 当前页码 |
| `sort` | `-updated_at` | `updated_at` 为升序，`-updated_at` 为降序 |

切换分类会重置页码；切换排序也会回到第一页。页码超过筛选结果的最后一页时，页面自动回到有效末页。

分类列表遵循 TB-V10-025 的直属分类语义，不自动合并后代分类文档。分类节点显示的数量来自分类树统计，用于反映该分类子树的知识规模。

## 4. 页面状态

- 加载中：分类树和文档区分别展示骨架屏。
- 请求失败：对应区域展示错误提示及重试操作。
- 分类为空：分类区显示“暂无分类”。
- 当前分类无内容：显示“该分类暂无可浏览文档”。
- 知识库无内容：显示初始化空状态，并提供同步管理入口。

为避免快速切换分类时旧请求覆盖新结果，文档请求使用递增序号，仅接收最后一次请求结果。

## 5. 验收结论

- 分类树保持父子层级，支持切换当前分类。
- 首页传入的 `category_id` 能直接恢复选中分类并加载对应文档。
- 切换分类后文档列表和标题同步更新。
- 分页与排序状态进入 URL，刷新不丢失。
- 全部知识、空分类和空知识库均具有明确空状态。
- 文档列表保持默认可见性与稳定更新时间排序。
