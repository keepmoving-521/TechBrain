# 标签知识列表页

本文档对应 TB-V10-028：实现标签知识列表页。

## 1. 页面入口

标签浏览复用知识库页面：

```text
/knowledge?view=tags
/knowledge?tag_id={tag_id}
```

传入 `tag_id` 时自动进入标签模式并选中目标标签，因此知识首页的常用标签入口可以直接跳转。页面支持在“分类”和“标签”两种浏览方式间切换。

## 2. 数据接口

```http
GET /api/v1/tags?page={tag_page}&page_size=30&sort=-usage_count
GET /api/v1/tags/{tag_id}
GET /api/v1/documents?tag_id={tag_id}&status=published,draft,archived,deprecated
```

标签按使用次数降序分页展示，使用次数相同时沿用后端稳定排序。标签详情用于恢复不在当前标签分页内的深链接名称和统计。

关联文档复用统一文档列表接口，因此支持稳定分页及源更新时间升序、降序排列。

## 3. 统计一致性

标签查询的 `usage_count` 统计全部未软删除文档，包括：

- `published`
- `draft`
- `archived`
- `deprecated`

标签模式向文档列表显式传入相同的四种状态，确保页面显示的关联文档总数与 `usage_count` 一致。草稿、归档和弃用文档会在列表中显示状态标识，软删除文档始终排除。

分类模式不受此规则影响，仍默认只展示 `published` 和 `deprecated` 文档。

## 4. URL 状态

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| `view` | `categories` | `tags` 表示标签浏览模式 |
| `tag_id` | 无 | 当前标签 ID |
| `tag_page` | `1` | 标签列表页码 |
| `page` | `1` | 关联文档页码 |
| `sort` | `-updated_at` | 关联文档更新时间排序 |

选择新标签会将关联文档重置到第一页；标签分页不会改变当前文档页码。切换文档排序会重置文档页码，但保留标签及标签页位置。

## 5. 页面状态

- 未选择标签：提示从左侧选择标签。
- 空标签：显示“该标签暂无关联文档”。
- 无标签：显示标签空状态。
- 请求失败：标签区和文档区分别提供重试操作。
- 快速切换：文档区只接收最后一次请求结果，避免旧标签结果覆盖当前标签。

## 6. 验收结论

- 标签列表展示名称及准确使用次数。
- 标签使用次数和关联文档分页总数采用同一统计口径。
- 支持标签分页、关联文档分页和更新时间排序。
- 首页 `tag_id` 入口能够恢复标签模式及选中状态。
- 空标签、未选择标签和请求失败均有明确页面状态。
