# 标签管理说明

本文档对应 TB-V10-023：实现标签管理。

## 1. 能力范围

标签管理提供：

- 新建未使用标签。
- 重命名标签，并安全回写所有关联 Markdown 文档。
- 删除未被任何文档使用的标签。
- 规范化名称、文件版本和运行任务冲突检测。

标签合并不属于本需求。新名称与其他标签冲突时不会自动合并，由 TB-V10-024 提供显式合并能力。

## 2. 新建标签

### 请求

```http
POST /api/v1/tags
Content-Type: application/json
```

```json
{
  "name": "ORM"
}
```

成功返回 `201 Created`：

```json
{
  "id": 1,
  "name": "ORM",
  "normalized_name": "orm",
  "status": "active",
  "usage_count": 0,
  "created_at": "2026-06-29T10:00:00+08:00",
  "updated_at": "2026-06-29T10:00:00+08:00"
}
```

新建标签尚未出现在任何 Markdown 中，因此只创建结构化标签，不修改知识源文件。

若规范化名称已经存在，返回 `409 Conflict`：

```text
标签规范化名称已存在: orm
```

唯一性覆盖大小写、全角/半角字符和多余空白差异。

## 3. 重命名标签

### 请求

```http
PATCH /api/v1/tags/{tag_id}
Content-Type: application/json
```

```json
{
  "name": "object-relational-mapping"
}
```

重命名保持标签 ID 不变，并更新：

- `knowledge_tags.name`。
- `knowledge_tags.normalized_name`。
- 所有关联 Markdown 的 Front Matter `tags`。
- Markdown `updated_at`。
- `knowledge_documents.tags` JSON 镜像。
- 文档标签结构化关联和 Front Matter 哈希。

示例：

```yaml
tags:
  - orm
  - performance
```

重命名后：

```yaml
tags:
  - object-relational-mapping
  - performance
```

标签只修改大小写时同样回写 Markdown，但仍保持原标签身份。

## 4. 安全回写流程

已使用标签重命名按以下流程执行：

1. 校验新名称并计算规范化名称。
2. 检查规范化名称没有被其他标签占用。
3. 加载标签关联的全部文档，包括软删除文档检查。
4. 确认源文件存在并位于配置的知识库内。
5. 重新解析 Markdown，对比文档 ID、正文哈希和 Front Matter 语义哈希。
6. 确认每篇文档恰好包含一个当前标签。
7. 生成新的 `tags` 和 `updated_at` Front Matter 内容并再次解析校验。
8. 先更新标签名称，再原子写入文件并重新同步文档。
9. 全部成功后提交数据库事务。

如果标签关联了已软删除文档，由于对应知识源文件不可保证存在，重命名会被拒绝，不会只更新数据库。

Front Matter 回写使用当前解析器支持的规范化列表格式。标签名称同时包含单引号和双引号时，已使用标签的重命名会被拒绝，避免生成无法可靠往返解析的 YAML 子集内容。

## 5. 删除标签

### 请求

```http
DELETE /api/v1/tags/{tag_id}
```

只有无任何文档关联的标签可以删除，成功返回：

```http
HTTP/1.1 204 No Content
```

标签只要仍存在任何关联，包括软删除文档关联，就返回：

```http
HTTP/1.1 409 Conflict
```

```text
标签正在被文档使用, 无法直接删除
```

删除接口不会隐式修改 Markdown，也不会从文档标签列表中静默移除标签。用户应先编辑 Markdown 并执行同步，或使用后续标签合并能力迁移关联。

## 6. 冲突与补偿

以下情况返回 `409 Conflict`：

- 新建或重命名后的规范化名称已存在。
- 标签正在被文档使用却尝试删除。
- 标签关联软删除文档却尝试重命名。
- Markdown 在最近同步后被外部修改。
- Markdown 在预检完成后、写入前再次变化。
- 文档中的标签与结构化关联不一致。
- Front Matter 无法安全序列化或重新解析。
- 知识同步或其他元数据写操作正在执行。

多文档重命名中途失败时：

- 数据库标签名称和文档关联整体回滚。
- 已写入 Markdown 按相反顺序恢复原内容。
- 未写入文件保持不变。
- 如果文件在补偿阶段再次被外部修改，错误会列出需要人工检查的文件。

## 7. 状态码

| 状态码 | 场景 |
| --- | --- |
| `200` | 标签重命名成功 |
| `201` | 标签创建成功 |
| `204` | 未使用标签删除成功 |
| `400` | 标签名称不符合领域规则 |
| `404` | 标签不存在 |
| `409` | 名称、使用状态、文件版本或运行任务冲突 |
| `422` | 请求字段缺失、类型错误或长度错误 |

## 8. 验收结论

TB-V10-023 已覆盖：

- 支持标签新建、重命名和删除。
- 重命名保持标签 ID 和文档关联身份稳定。
- 已使用标签重命名安全回写全部关联 Markdown。
- 外部修改和多文件失败时不会产生数据库与 Markdown 双重事实。
- 规范化名称冲突不会静默覆盖或合并。
- 被任何文档使用的标签不可直接删除。
- 未使用标签可以安全删除。
