# 标签合并说明

本文档对应 TB-V10-024：实现标签合并。

## 1. 设计目标

标签合并用于处理同义、近义或历史命名不一致的标签，将源标签的全部文档归并到一个现有目标标签。

合并遵循：

- 目标标签必须显式指定，不根据名称自动猜测。
- 所有源标签文档安全回写 Markdown。
- 文档最终只保留一个目标标签关联。
- 源标签不物理删除，状态改为 `archived`。
- 目标标签自动激活为 `active`。
- 任何冲突或文件失败都会整体回滚。

## 2. API

### 请求

```http
POST /api/v1/tags/{source_tag_id}/merge
Content-Type: application/json
```

```json
{
  "target_tag_id": 12
}
```

### 成功响应

```json
{
  "source_tag_id": 8,
  "target_tag_id": 12,
  "migrated_document_count": 3,
  "source_status": "archived"
}
```

`migrated_document_count` 表示合并前直接关联源标签的文档数量。源标签没有文档时仍可合并，数量为 `0`，源标签归档、目标标签激活。

## 3. 文档标签变换规则

### 3.1 文档只包含源标签

合并 `orm` 到 `database`：

```yaml
tags:
  - orm
  - performance
```

更新为：

```yaml
tags:
  - database
  - performance
```

源标签在原位置替换为目标标签。

### 3.2 文档同时包含源标签和目标标签

```yaml
tags:
  - orm
  - database
  - performance
```

更新为：

```yaml
tags:
  - database
  - performance
```

系统只删除源标签，不再追加目标标签。因此：

- Front Matter 不产生重复目标标签。
- `knowledge_document_tags` 联合主键不会发生重复冲突。
- 文档与目标标签只保留一条关联。

### 3.3 目标标签原为归档状态

只要目标标签被选为合并目标，就会更新为：

```text
status = active
```

源标签完成迁移后更新为：

```text
status = archived
```

## 4. 安全合并流程

1. 检查源标签和目标标签存在且不是同一标签。
2. 加载源标签关联的全部文档。
3. 若包含软删除文档，拒绝合并，避免无法回写知识源。
4. 检查每个 Markdown 文件仍在知识库内且可解析。
5. 对比文档 ID、正文哈希和 Front Matter 语义哈希，检测外部修改。
6. 对每篇文档计算替换或去重后的标签集合。
7. 校验新标签集合不存在规范化重复。
8. 更新源、目标标签状态并原子写入 Markdown `tags`、`updated_at`。
9. 重新同步每篇文档，迁移结构化标签关联。
10. 确认源标签已不存在任何文档关联后提交事务。

## 5. 源标签保留策略

合并后源标签选择归档而不是物理删除：

- 保留历史标签身份，便于审计。
- 阻止旧名称被误创建为新标签。
- 为后续合并历史、操作记录和恢复能力保留基础。
- 查询接口仍可查看源标签，但 `usage_count` 为 `0`、状态为 `archived`。

如果确认不再需要源标签，可以在合并完成后通过标签删除接口删除这个未使用标签。

## 6. 冲突处理

以下情况返回 `409 Conflict`：

- 源标签和目标标签相同。
- 源标签关联软删除文档。
- Markdown 在最近同步后被外部修改。
- 文档标签和结构化关联不一致。
- 合并后的 Front Matter 标签无法安全序列化或解析。
- 文档重新同步后源标签仍存在关联。
- 知识同步或其他元数据写操作正在执行。

源标签或目标标签不存在时返回 `404`，错误信息分别为：

```text
源标签不存在
目标标签不存在
```

## 7. 事务与补偿

多文档合并中任一步骤失败时：

- 源标签恢复原状态。
- 目标标签恢复原状态。
- 文档标签关联恢复到合并前。
- 已写入的 Markdown 按相反顺序恢复原内容。
- 未写入文件保持不变。

如果补偿期间文件再次被外部修改，系统不会覆盖外部内容，并返回需要人工检查的文件列表。

## 8. 验收结论

TB-V10-024 已覆盖：

- 支持将源标签的全部文档迁移到目标标签。
- 文档只含源标签时正确替换为目标标签。
- 文档同时包含源和目标时正确去重。
- 不产生重复 Front Matter 标签或重复数据库关联。
- 目标标签自动激活。
- 源标签关联清空并安全归档。
- 多文档失败时标签状态、关联和 Markdown 整体回滚。
