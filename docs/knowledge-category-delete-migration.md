# 分类删除与文档迁移说明

本文档对应 TB-V10-019：实现分类删除与文档迁移。

## 1. 设计目标

分类删除遵循显式迁移、禁止级联和不丢失归属原则：

- 有直属文档的分类不能直接删除。
- 有子分类的分类不能直接删除。
- 删除前可以把直属文档批量迁移到另一个现有分类。
- 文档迁移必须先安全回写 Markdown，再更新数据库关联。
- `knowledge_documents.category_id` 始终保持非空，不产生无分类文档。

## 2. 文档迁移接口

### 请求

```http
POST /api/v1/categories/{source_category_id}/documents/migrate
Content-Type: application/json
```

```json
{
  "target_category_id": 12
}
```

### 成功响应

```json
{
  "source_category_id": 8,
  "target_category_id": 12,
  "migrated_count": 3
}
```

迁移范围只包括直接归属于源分类的文档，不递归迁移子分类中的文档，也不改变子分类结构。这避免一次文档迁移隐式重组整棵分类树。

源分类没有直属文档时接口正常返回，`migrated_count` 为 `0`。

## 3. Markdown 回写

文档从 `backend/python` 迁移到 `database/mysql` 时，系统将 Front Matter 更新为：

```yaml
category: database/mysql
updated_at: 2026-06-29T12:00:00+00:00
```

回写流程复用分类管理的安全写入机制：

1. 校验源分类和目标分类存在且不同。
2. 加载源分类的全部直属文档。
3. 检查文档未被软删除，源文件仍在知识库内。
4. 对比文档 ID、原分类、正文哈希和 Front Matter 语义哈希。
5. 原子回写每个文件的 `category` 和 `updated_at`。
6. 重新同步文档，将 `category`、`category_id` 和哈希更新到数据库。
7. 全部成功后提交数据库事务。

当前策略与 TB-V10-018 一致：更新 Front Matter 分类，不自动移动物理 Markdown 文件，避免破坏相对链接和资源引用。

## 4. 分类删除接口

### 请求

```http
DELETE /api/v1/categories/{category_id}
```

空叶子分类删除成功返回：

```http
HTTP/1.1 204 No Content
```

删除前依次检查：

- 是否仍有任何直属文档，包括软删除文档。
- 是否仍有任何直属子分类。
- 是否存在数据库层面的其他引用。

分类删除不会执行以下隐式操作：

- 不级联删除文档。
- 不级联删除子分类。
- 不自动选择目标分类。
- 不自动把子分类提升到父级。

## 5. 推荐操作流程

有文档、无子分类：

```text
迁移直属文档 → 确认 migrated_count → 删除空分类
```

有文档、有子分类：

```text
迁移直属文档
→ 分别移动、清空或删除子分类
→ 删除当前空分类
```

## 6. 冲突与错误

| 状态码 | 场景 |
| --- | --- |
| `404` | 源分类、目标分类或待删除分类不存在 |
| `409` | 源目标相同、文档被外部修改、存在软删除文档、分类仍有文档或子分类 |
| `422` | `target_category_id` 缺失或类型不合法 |

典型删除冲突：

```json
{
  "error": {
    "code": "HTTP_409",
    "message": "分类仍包含文档, 请先迁移文档",
    "details": null
  },
  "request_id": "..."
}
```

```json
{
  "error": {
    "code": "HTTP_409",
    "message": "分类仍包含子分类, 无法直接删除",
    "details": null
  },
  "request_id": "..."
}
```

## 7. 事务与补偿

多文档迁移中任一步骤失败时：

- 数据库事务整体回滚。
- 已写入的 Markdown 文件恢复原始内容。
- 未写入文件保持不变。
- 所有文档继续关联原分类。

迁移和删除与知识同步、分类管理共用同一运行锁，同一进程内不会重叠修改知识文件和分类数据。

## 8. 验收结论

TB-V10-019 已覆盖：

- 支持将分类直属文档批量迁移到指定目标分类。
- 迁移安全回写 Markdown 并更新结构化分类关联。
- 迁移后所有文档仍具有有效非空分类外键。
- 有文档或子分类时分类不可直接删除。
- 只有无文档、无子分类的叶子分类可以删除。
- 多文件迁移失败时文件和数据库均回滚。
